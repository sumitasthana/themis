# Project Docs Module (`docs/`)

## Purpose
The `docs/` module stores curated product and engineering reference material separate from executable runtime modules.

## Current Contents
- `CHANGELOG.md` — milestone and phase history.
- `DESIGN_SYSTEM.md` — visual system and UI patterns.
- `QUICK_REFERENCE.md` — quick lookup tokens and snippets.
- `themis_data_doc.docx` — data model/reference artifact.

## How This Module Is Used
- Supports onboarding for engineering and product contributors.
- Provides non-code policy and design context for implementation tasks.
- Acts as durable reference alongside code-level wiki docs.

## Relationship to `wiki/`
- `docs/` contains product/process artifacts.
- `wiki/` contains module-level technical implementation documentation.
- Keep both synchronized when major architecture or behavior changes occur.
